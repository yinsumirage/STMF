STMF / sensor-guided temporal MANO refinement

近期实验进度展示：从 plain HaMeR 调参转向 sensor-guided temporal refiner

当前实验主线不再是继续硬调 HaMeR 在 HO3D-v3 上的单帧 finetune 分数，而是固定 HaMeR base prediction， 检验低维拉线 pseudo-sensor 是否能在时序 MANO refine 中提供额外约束。

重要解释：当前 HO3D-v3 上 base HaMeR 的 self-eval PA-MPJPE 约为 18.920mm， 明显高于你预期的 10mm 左右，也高于早期约 13mm 的结果。这个绝对值现在不能简单解释成“模型太差”或“训练不够”， 更可能混合了 v2/v3 protocol 差异、evaluation split 语义、GT/关节顺序、camera/patch 评测协议和 base cache 质量等因素。

2. 当前 pipeline

01 打包 GTHO3D-v3 train/eval NPZ，包含 MANO、3D joints、imgname、bbox 等。
02 缓存 HaMeR离线跑 HaMeR checkpoint，生成每帧 base_pose/base_cam。
03 构造时序窗口每个 target frame 带过去若干帧 pose window、sensor window 和 valid mask。
04 训练 refiner只预测 hand_pose residual，默认不改 global orient / camera。
05 stateful eval逐序列评测，把 refined pose 回填历史，计算 3D 和 temporal 指标。

7. 为什么 HO3D-v3 绝对分数偏高

当前 refiner 的结果不能解释为“HO3D-v3 应该就是 18mm”。更合理的说法是：在当前本地 self-eval protocol 下， raw HaMeR cache 已经是 18.920mm，因此 refiner 的所有收益都建立在这个 base 上。绝对分数偏高需要单独排查。
可能来源为什么会影响 PA-MPJPE / PA-MPVPE当前判断
HO3D v2 / v3 protocol 差异 论文里常见结果多来自 v2 或官方协议，v3 标注和 split 语义可能不同。 需要回到 v2 protocol 或官方 server/官方格式再确认。
Evaluation NPZ 是否等价官方 evaluation 当前 ho3d_evaluation.npz 有 GT 并做 self-eval，未必和论文里的官方提交协议完全一致。 这是最需要单独隔离的 protocol 问题。
关节顺序 / MANO / handTrans / camMat HO3D 官方顺序、HaMeR/OpenPose 顺序、MANO 重建顺序混用会直接抬高 3D 指标。 之前做过一致性排查，但只要重新导数据就要重跑检查。
Camera / patchK / bbox scale HaMeR 原本依赖 crop/patch 相机近似，真实 camera intrinsics 差异会造成 2D/3D 监督拉扯。 已作为 negative result / protocol analysis 保留。
Refiner 没有重训 RGB backbone 当前 v2 只修 cached pose residual，不会从图像重新恢复更好的 global/camera 信息。 解释了为什么能改善但不能把 18mm 拉回 10mm。
Optimizer step 数 20 epoch 全量覆盖 train split，但 batch 很大，optimizer step 约 220。 可能影响上限；但更长 epoch 已经出现过拟合迹象，应尝试小 batch 多 step 而不是只加 epoch。

8. 现在能证明什么，不能证明什么

能证明

  • 在固定 HaMeR cache 的 HO3D-v3 protocol 下，temporal refiner 比 raw HaMeR 更好。
  • 同一结构下，sensor-guided refiner 比 zero-sensor pose-only refiner 更好。
  • sensor 收益不是简单 EMA 平滑带来的。
  • sensor 对 base pose noise 和 moderate sensor noise 有一定鲁棒性。

还不能证明

  • 不能证明真实拉线手套数据不经校准就可直接用。
  • 不能证明真实遮挡视频上一定恢复更好。
  • 不能证明换 WiLoR / AnyHand backbone 后结论完全相同。
  • 不能解释 HO3D-v3 绝对分数为什么从你预期的 10mm 左右升到 18mm。

9. 给老师汇报时可以怎么说

前期继续直接 finetune HaMeR 在 HO3D-v3 上提分不稳定，而且 hard case 中存在 2D/3D/MANO/camera protocol 拉扯。 因此现在把主线调整为 sensor-guided temporal MANO refinement： 固定 HaMeR 作为 RGB base，使用公开 benchmark 上由 GT 生成的 pseudo-sensor 验证低维拉线先验是否能稳定 finger articulation。

当前全量 HO3D-v3 cached 实验表明，sensor-guided refiner 在 clean、base pose noise、sensor dropout/noise 下都稳定优于 pose-only refiner， 并且优于 HaMeR+EMA 的 primary 3D 指标。下一步需要补两件事：一是继续隔离 HO3D-v3 绝对分数偏高的 protocol 来源； 二是把真实拉线数据校准到同一套 (T, 5) sensor 接口，做实时 demo 和真实遮挡场景展示。
